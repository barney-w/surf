**Context**: The app needs to detect whether it's running inside a Tauri desktop shell vs a regular browser. Tauri injects `__TAURI_INTERNALS__` into the webview's global scope. This detection is used to choose between popup (desktop) and redirect (web) auth flows.

## Milestone 4: Security Hardening

### Task 4.1: Configure CSP and Security in tauri.conf.json
**Context**: The desktop app needs a strict Content Security Policy. The app loads Google Fonts from external URLs (see `web/index.html` lines 4-6), calls Microsoft login endpoints, the Surf API, and Microsoft Graph API. The CSP must allow these while blocking everything else.

Current `index.html` loads fonts from:
- `https://fonts.googleapis.com` (CSS)
- `https://fonts.gstatic.com` (font files)

The app connects to:
- Microsoft login: `https://login.microsoftonline.com`, `https://*.login.microsoftonline.com`
- Microsoft Graph: `https://graph.microsoft.com`
- Surf API: configurable via `VITE_DESKTOP_API_URL` (varies per deployment)
- API proxy in dev: `http://localhost:8090`

**Files to modify**:
- `web/src-tauri/tauri.conf.json` — add CSP and security settings

**Specification**:

Add to the `app` section of `tauri.conf.json`:
```json
{
  "app": {
    "security": {
      "csp": "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; font-src 'self' data: https://fonts.gstatic.com; img-src 'self' blob: data: https:; connect-src 'self' https://login.microsoftonline.com https://*.login.microsoftonline.com https://graph.microsoft.com https://*.azurecontainerapps.io; frame-src https://login.microsoftonline.com",
      "freezePrototype": true
    }
  }
}
```

Notes on the CSP:
- `style-src` includes `'unsafe-inline'` because Tailwind CSS and React inject inline styles
- `style-src` includes `https://fonts.googleapis.com` for Google Fonts CSS
- `font-src` includes `https://fonts.gstatic.com` for Google Fonts files
- `connect-src` includes `https://*.azurecontainerapps.io` as a wildcard for Azure Container Apps (the actual API domain). Deployments with custom domains should replace this with the specific domain.
- `frame-src` allows Microsoft login for MSAL popup/iframe flows
- `freezePrototype: true` prevents prototype pollution attacks

### Task 5.2: Add Offline Detection Banner & Update Checker
**Context**: The app requires network connectivity (it's a chat interface to a remote API). We should handle offline states gracefully with a banner. We should also check for updates on app launch using the Tauri updater plugin (already installed in Milestone 1).

**Specification**:

1. Add an `OfflineBanner` component to `App.tsx` that shows when the browser is offline:
   ```typescript
   function OfflineBanner() {
     const [isOnline, setIsOnline] = useState(navigator.onLine);

     useEffect(() => {
       const handleOnline = () => setIsOnline(true);
       const handleOffline = () => setIsOnline(false);
       window.addEventListener('online', handleOnline);
       window.addEventListener('offline', handleOffline);
       return () => {
         window.removeEventListener('online', handleOnline);
         window.removeEventListener('offline', handleOffline);
       };
     }, []);

     if (isOnline) return null;

     return (
       <div className="bg-amber-500/90 text-white text-sm text-center py-1.5 px-4">
         You're offline. Reconnect to continue chatting.
       </div>
     );
   }
   ```

2. Add an update check effect inside `AppContent`. Import `isTauri` from `./auth/platform` and add:
   ```typescript
   useEffect(() => {
     if (!isTauri()) return;

       check().then((update) => {
         if (update) {
           if (window.confirm(`Version ${update.version} is available. Update now?`)) {
             void update.downloadAndInstall();
           }
         }
       }).catch(() => {
         // Silent fail — don't block app usage
       });
     }).catch(() => {
       // Plugin not available
     });
   }, []);
   ```
   Use dynamic import so the plugin doesn't break web builds (it's only available in Tauri context).

